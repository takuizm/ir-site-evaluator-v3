# LLM 観点別テンプレート

LLMValidator が観点ごとにブレない指示を出せるよう、カテゴリ/サブカテゴリ別のヒント文言を定義する。`src/validators/llm_validator.py` の `CATEGORY_HINTS` に対応。

|カテゴリ|サブカテゴリ|ヒント例|備考|
|---|---|---|---|
|ウェブサイトの使いやすさ|メニューとナビゲーション|"メニューの見出しやリンク文言を引用し、観点の要件（例: 同一デザイン・現在位置表示・横移動可否）と一致する根拠がある場合のみ PASS としてください"|anchor text の確認・位置特定を促す|
|ウェブサイトの使いやすさ|デザインとアクセシビリティ|"色・フォント・レイアウトなど視覚的要素を本文から抜き出し、観点に必要な UI が存在するか判断してください。曖昧な表現は FAIL とします"|VISUAL 依存が強い観点を LLM から切り離す準備|
|企業・経営情報の充実度|事業内容と経営戦略|"中期経営計画・ビジョン・定量目標など経営方針に関する記述を引用し、観点どおりの内容がある場合のみ PASS"|KPI/戦略系項目用|
|企業・経営情報の充実度|ESGへの取り組み|"サステナビリティ/ESG セクションの記述から、施策・KPI・ガバナンス構造など観点に必要な情報を提示してください"|ESG 項目での逸脱抑制|
|情報公開の透明性|IR資料|"IR資料のタイトルと公開形態（HTML/PDF、期間等）を列挙し、観点の要件（最新年度、有価証券報告書、ハイライト等）に合致する場合のみ PASS"|資料掲載系テンプレ|

## 実装メモ
- `CATEGORY_HINTS` は `(category, subcategory)` キー、`CATEGORY_ONLY_HINTS` は `category` のみで適用。
- 更に item 固有のヒントは `ITEM_HINTS`（辞書）で上書きする。
- ヒントは `build_prompt()` 内の `## 追加要件` に `-` 箇条書きで出力。
- 重複ヒントは set で除外し、最大 4 件程度に制限。

このドキュメントはカテゴリ追加時に更新する。
